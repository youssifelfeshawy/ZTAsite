Zero Trust Architecture (ZTA)

This project involves setting up a simulated secure network using virtual machines (VMs) in VirtualBox. The focus is on creating isolated environments for users, servers, and authentication, with all traffic routing through a central gateway for enhanced security.

Stage 1 Overview: Building the Virtual Machines

In this stage, we establish the foundational virtual infrastructure:

  • Host Environment: Install Oracle VirtualBox (v7.1.10) and allocate resources (4GB RAM, 3 CPUs per VM; storage varies by role).
  • User VMs: 1-2 lightweight Linux Mint VMs for simulating user access, connected to an internal "Users" network.
  • Main Server VM: Ubuntu Server hosting a web app (LAMP stack) and SFTP for secure file access, on "Main" network.
  • Gateway VM: Ubuntu Server acting as router with 4 adapters (bridged for internet, internal for segments), handling DHCP, IP forwarding, and traffic monitoring with Wireshark.
  • Authentication VM: Ubuntu Server for identity management, on "Auth" network.
  • Network Setup: Isolated subnets (Users: 192.168.1.0/25, Main: 192.168.1.128/30, Auth: 192.168.1.132/30). All traffic routes through gateway; IPv6 disabled.
  • Testing: Verify connectivity, isolation, and monitor traffic.

The goal is to simulate a secure environment where VMs communicate only via the gateway, preventing direct access and enabling security testing.
